EVENT(1)

NAME

event - manage named shell event hook registrations.

SYNOPSIS

event add event name function [--priority number]
event add timer.tick name function --every milliseconds [--priority number]
event remove event name
event list [event]

DESCRIPTION

event registers shell functions to run when Rush observes shell events. Registrations are named so configuration can replace or remove one hook without disturbing others. If an event add command uses an existing event and name, it replaces that registration.

Hooks run in ascending priority order, then by registration name. The default priority is zero. Hook execution preserves the visible last status and pipeline statuses unless the hook exits the shell.

EVENTS

directory.change
Runs after a successful current-shell directory change has committed. The hook function receives the old directory as $1 and the new directory as $2.
prompt.prepare
Runs on the real interactive shell state before Rush renders each prompt.
timer.tick
Runs while the interactive line editor is idle after the registration's --every interval. The hook function receives the registration name as $1.

EXAMPLE

sync_project_env() {
  if test -f .envrc; then
    . ./.envrc
  fi
}

remember_prompt_cwd() {
  rush_prompt_cwd="$PWD"
}

event add directory.change project-env sync_project_env --priority -10
event add prompt.prepare prompt-cwd remember_prompt_cwd
event add timer.tick clock remember_prompt_cwd --every 1000

EXIT STATUS

Zero on success and two for usage errors.
